# C — 김세아 · 여전히 빈 집에 있습니다

전시용 영상·라디오 수신기 웹앱이다. 영상과 라디오는 서로 독립적으로 순환하며, 같은 영상도 매번 다른 라디오와 만날 수 있다. 앱은 외부 API, CDN, 스트리밍, 외부 폰트 없이 빌드 후 완전히 오프라인으로 실행된다.

## Windows에서 가장 짧게 실행

Node.js 22 이상과 pnpm을 설치한 뒤 프로젝트 폴더에서 다음을 실행한다.

```powershell
pnpm install
pnpm dev
```

브라우저에 표시된 `http://localhost:...` 주소를 Chrome/Edge에서 열고 **START RECEIVER**를 누른다. `file://`로 직접 열면 manifest `fetch`가 CORS 정책에 막히므로 반드시 localhost를 사용한다.

macOS도 동일하게 `pnpm install`, `pnpm dev`를 사용한다. 완성 빌드는 `pnpm build`, 로컬 실행은 `pnpm start`다. 한번 설치와 빌드를 마친 뒤에는 모든 앱 리소스가 장치에 있어 인터넷이 필요 없다.

## 조작

- `↑` / `↓`: 이전 / 다음 영상
- `←` / `→`: 이전 / 다음 라디오
- 화면 버튼: 마우스·터치 조작
- `FULL SCREEN`: 전체화면 요청
- `D`: 개발 진단 화면

브라우저 자동재생 정책 때문에 최초 한 번의 클릭/터치가 필요하다. Wake Lock은 지원 브라우저에서만 요청되며 실패해도 재생은 계속된다.

## 실제 미디어 넣기

- 영상 10개: `public/media/videos/video_001.mp4`부터 `video_010.mp4`
- 라디오 5개: `public/media/radio/radio_001.mp3`부터 `radio_005.mp3`
- 튜닝 효과: `public/media/effects/`

저작권 미디어를 포함하지 않았으므로 현재는 화면 신호 플레이스홀더가 보인다. H.264/AAC MP4와 MP3가 Chrome/Chromium 호환성이 가장 좋다. 사진도 사운드와 합친 MP4로 미리 렌더한다.

개수를 바꾸려면 파일을 추가/제거하고 `public/data/videos.json` 또는 `radio.json` 항목만 같은 순서로 수정한다. 재생 코드는 배열 길이를 자동 사용한다. `pnpm validate:media`는 JSON과 파일 존재 여부를 확인하고, `pnpm generate:manifests`는 폴더 스캔 결과를 초안 JSON으로 출력한다.

## 설정

`src/config.ts`에서 흰 화면 300ms, 각 볼륨, 라디오 페이드와 튜닝 시간, 최소 잔여 재생 시간, 컨트롤/전시/진단 모드, Wake Lock, 상태 복원을 조정한다. 전시 모드는 화면 컨트롤과 진단을 숨기도록 배포 전에 설정한다.

## 1024×600 및 장시간 시험

Chrome 개발자 도구의 반응형 모드에서 1024×600으로 확인한 뒤 실제 7인치 패널에서 전체화면을 시험한다. 가속 시험은 복제한 개발 설정에서 전환 시간을 줄이고 선택적 자동 라디오 전환을 켜 수백 회 수행한다. 일반 전시 설정에는 가속 값을 넣지 않는다. 최종 미디어로 9시간 이상 연속 실행하면서 작업 관리자 메모리, 오류 목록, 영상/오디오 정지, Pi 온도를 기록한다.

## Windows/macOS 화면 절전

전원 설정에서 디스플레이 끄기와 절전을 `안 함`으로 설정하고 브라우저 알림을 끈다. macOS는 전원 어댑터 연결 중 자동 잠자기를 비활성화한다. Wake Lock만을 유일한 방지책으로 의존하지 않는다.

## Raspberry Pi 3 이전

Raspberry Pi OS에 정적 서버 systemd 서비스를 만들고 Chromium을 키오스크 모드로 자동 시작한다. H.264 1024×600 영상 하나와 오디오 하나를 동시에 재생하는 목표는 Pi 3 실기에서 프레임 드롭과 온도를 측정한 뒤 보드 교체 여부를 판단한다. GPIO 또는 ESP32-C3 브리지는 `docs/ARCHITECTURE.md`의 네 가지 명령을 WebSocket으로 보낸다. 폐장 시 콘센트를 뽑지 않고 물리 종료 버튼으로 안전 종료한다. 예비 microSD, 전체 이미지 백업, 자동 재시작과 읽기 전용 루트 적용을 최종 안정화 단계에서 검증한다.

## 알려진 제약

브라우저는 로컬 폴더를 열람할 수 없어 manifest가 필요하다. 코덱 미지원·손상 파일은 재생할 수 없으며 진단에 기록된다. 오디오 무작위 시작점은 metadata가 로드된 뒤에만 정확하다. 실제 GPIO/TFT 펌웨어, 분리 오디오 출력, 튜닝 효과 파일 선택은 하드웨어 모델과 실제 음원이 확정된 뒤 마무리한다.
